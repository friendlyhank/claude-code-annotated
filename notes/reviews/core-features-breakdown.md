1. 架构设计和核心流程
 - 分层边界（交互层、查询引擎层、工具执行层、API 层、会话层、TUI 层）
 - 主执行链路（输入 -> queryLoop -> tool_use -> runTools -> next_turn/终止）
 - 文档阅读路径（先总览，再按能力域下钻）

2. 核心交互层
 - CLI 入口与参数解析
 - REPL 循环机制
 - 用户输入捕获与路由
 - 命令解析系统

3. 查询引擎层
 - State 状态管理（消息历史、工具上下文、追踪状态）
 - `query()` 生成器与 `queryLoop` 主循环
 - 流式响应消费与 `tool_use` 检测
 - 工具结果回灌与轮次推进（`next_turn`）
 - 中断与错误恢复（`AbortController`、缺失工具结果补偿）

4. 工具执行层
 - 工具注册与查找（`Tool`/`Tools`、`findToolByName`）
 - 工具编排（按并发安全性分批、串行/并发调度）
 - 单工具执行入口（`runToolUse`）
 - 执行状态标记（in-progress 标记与完成清理）
 - 工具结果消息构造与回传

5. API 客户端层
 - 模型调用抽象（`QueryDeps.callModel`）
 - 流式响应处理
 - 错误重试与恢复入口（待逐步对齐）
 - 计费与用量统计接口（待逐步对齐）

6. 会话管理层
 - 启动态与全局状态管理
 - 会话历史与配置承载
 - 跨轮次状态同步

7. TUI 渲染层
 - Ink 组件系统
 - 交互反馈（进度、状态）
 - 快捷键绑定
 - 样式与主题
