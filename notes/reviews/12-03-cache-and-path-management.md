# 12.3 缓存与路径管理

## 概览

缓存与路径管理模块提供标准化的缓存目录管理和路径安全处理，支持项目隔离和跨平台兼容。

## 核心模块

### cachePaths.ts - 缓存路径管理

**职责**：基于 env-paths 提供标准缓存目录

**缓存类型**：
- 错误日志缓存
- 消息缓存
- MCP 日志缓存

**项目隔离**：
- 按 cwd 生成隔离的缓存目录
- 不同项目缓存互不干扰

**路径安全处理**：
- 非法字符替换
- 长度限制
- 哈希后缀（避免冲突）

### path.ts - 路径工具函数

**职责**：路径操作和转换

**导出函数**：
- `expandPath()`：路径展开（`~` 展开、环境变量展开）
- `toRelativePath()`：相对路径转换
- 路径规范化

### cwd.ts - 工作目录管理

**职责**：工作目录访问和管理

**导出函数**：
- `getCwd()`：获取当前工作目录
- `setCwd()`：设置工作目录

**设计原因**：
- 统一工作目录访问入口
- 支持测试时 mock

## 缓存目录结构

```
~/.cache/claude-code/
  ├── <project-hash-1>/
  │   ├── errors/
  │   ├── messages/
  │   └── mcp-logs/
  └── <project-hash-2>/
      ├── errors/
      ├── messages/
      └── mcp-logs/
```

## 路径安全策略

### 非法字符处理

不同平台对文件名有不同限制：
- Windows：`< > : " / \ | ? *`
- macOS/Linux：主要是 `/` 和 null

**处理方式**：替换为安全字符

### 长度限制

- Windows：MAX_PATH = 260 字符
- macOS/Linux：通常 255 字符（文件名）

**处理方式**：超长路径截断并添加哈希后缀

### 项目隔离哈希

```typescript
项目路径 → SHA256 哈希 → 取前 8 字符 → 缓存目录名
```

## 调用关系

```
工具层
    ↓
cachePaths.ts (获取缓存路径)
    ↓
path.ts (路径处理)
    ↓
cwd.ts (工作目录)
```

## 设计亮点

1. **标准路径**：使用 env-paths 遵循各平台惯例
2. **项目隔离**：不同项目缓存分离，避免冲突
3. **路径安全**：自动处理非法字符和长度限制
4. **可测试性**：工作目录访问统一入口，便于 mock
