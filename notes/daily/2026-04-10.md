10. 实现原理与验证方式

- 实现原理：
- `REPL` 在提交时先把用户消息写入本地 transcript，再以当前消息快照构建最小 `ToolUseContext`
- `REPL` 使用与上游一致的消费方向，直接迭代 `query()` 生成器，而不是手写另一套“聊天处理器”
- `query/deps.ts` 的 mock `callModel()` 改为读取最后一条 `user` 消息并生成 assistant 文本，证明当前 UI 响应来自代理循环而不是本地分支
- `replLauncher.tsx` 保持与上游一致的窄职责，只负责把 `App` 和 `REPL` 组装后交给 `renderAndRun()`
- 验证方式：

```bash

bun run build
```
