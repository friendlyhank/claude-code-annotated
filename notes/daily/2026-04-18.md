## 实现原理与验证方式

**实现原理**：
- BashTool 通过 buildTool 工厂创建，定义 inputSchema（command/timeout/description/run_in_background）和 outputSchema（stdout/stderr/interrupted/exitCode）
- call() 是主入口：调用 exec() 获取 ShellCommand，await 其 result，然后通过 interpretCommandResult() 解释退出码
- exec() 使用 child_process.spawn 执行命令，通过 ShellCommand 抽象管理生命周期
- interpretCommandResult() 按命令类型判断退出码是否表示错误（如 grep 返回 1 表示无匹配而非错误）
- ShellError 区分 Shell 执行错误与其他错误，携带退出码和中断状态

**验证方式**：
- 输入 `command: "echo hello"`，触发 BashTool.call()，输出 "hello"
- 输入 `command: "grep pattern nonexistent"`，返回 exit code 2（真正的错误）
- 输入 `command: "grep pattern file_with_no_match"`，返回 exit code 1 但不视为错误

3. ShellCommand 的完整版有 TaskOutput、StreamWrapper、size watchdog 等复杂机制，简单版只做内存缓冲
