10. 实现原理与验证方式

- 实现原理：
- 将“用户提交”和“代理循环消费”拆成稳定边界，避免继续把所有逻辑堆在 `handleSubmit`
- 保持现有最小闭环行为不变：用户消息入列、消费 `query()` 产物、终态 `model_error` 时回写错误文本
- 验证方式：

```bash

bun run build
bun --eval "import { query } from './src/query.ts'; import { asSystemPrompt } from './src/utils/systemPromptType.ts'; const toolUseContext={ options:{ commands:[], debug:false, mainLoopModel:'claude-sonnet-4-6', tools:[], verbose:false, isNonInteractiveSession:false }, abortController:new AbortController(), getAppState:()=>({}), setAppState:()=>{}, setInProgressToolUseIDs:()=>{}, setResponseLength:()=>{}, updateFileHistoryState:()=>{}, updateAttributionState:()=>{}, messages:[] }; const iterator=query({ messages:[{ type:'user', uuid:crypto.randomUUID(), message:{ role:'user', content:'ping' } }], systemPrompt:asSystemPrompt(['test']), userContext:{}, systemContext:{}, canUseTool:async()=>true, toolUseContext, querySource:'repl' }); while (true) { const step=await iterator.next(); if (step.done) { console.log('TERMINAL', step.value?.reason ?? ```
```
