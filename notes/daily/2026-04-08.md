## 实现原理

### queryLoop 核心流程

1. **状态解构**：每次迭代开始解构 State，保持 bare-name 访问
2. **消息预处理**：构建 fullSystemPrompt，准备 messagesForQuery
3. **API 调用**：调用 deps.callModel() 获取流式响应
4. **流式处理**：for await 遍历响应，yield 中间状态
5. **工具检测**：提取 tool_use blocks，设置 needsFollowUp
6. **循环判断**：根据 needsFollowUp 决定继续或终止

### ModelCallParams 类型

```typescript
type ModelCallParams = {
  messages: Message[]
  systemPrompt: SystemPrompt
  signal: AbortSignal
  options: {

    isNonInteractiveSession: boolean
    [key: string]: unknown
  }
}
```
