10. 实现原理与验证方式

- 实现原理：
- `partitionToolCalls()` 先按 `isConcurrencySafe()` 把工具请求切成并发安全批次或单独串行批次
- `runTools()` 保持与上游一致的批次循环结构，并在每个更新点返回 `message + newContext`
- `queryLoop` 在检测到 `tool_use` 后不再终止，而是消费 `runTools()` 产物，把 `tool_result` 追加到 `toolResults`，再以 `next_turn` 更新 `state`
- 当前 `runToolUse()` 仍是最小 stub，但主链路控制流已经从“检测到工具就退出”升级为“检测到工具后进入下一轮”
- 验证方式：

```bash
bun run typecheck
bun run build
```
